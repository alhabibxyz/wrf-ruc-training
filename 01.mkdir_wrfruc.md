# 01. Create the WRF-RUC Working Directory

## Objective

Create the working directory and prepare the required WRF, WPS, WRFDA, and ARWpost environments by linking the pre-compiled executables and supporting files.

---

## Step 1. Create the Working Directory

Create the main working directory and its subdirectories.

```bash
mkdir WRFRUC_training
cd WRFRUC_training

mkdir run_wrfruc
mkdir ARWpost
mkdir asimilasi_d01
```

The directory structure should look like:

```
WRFRUC_training/
├── run_wrfruc/
├── ARWpost/
└── asimilasi_d01/
```

---

## Step 2. Prepare the WRF and WPS Working Directory

Move into the **run_wrfruc** directory and create symbolic links to the compiled WRF and WPS executables.

```bash
cd run_wrfruc

ln -sf /scratch/dmu/installer_wrf/WRF/run/* .
ln -sf /scratch/dmu/installer_wrf/WPS/link_grib.csh
ln -sf /scratch/dmu/installer_wrf/WPS/geogrid
ln -sf /scratch/dmu/installer_wrf/WPS/geogrid/geogrid.exe
ln -sf /scratch/dmu/installer_wrf/WPS/metgrid
ln -sf /scratch/dmu/installer_wrf/WPS/metgrid/metgrid.exe
ln -sf /scratch/dmu/installer_wrf/WPS/ungrib
ln -sf /scratch/dmu/installer_wrf/WPS/ungrib/ungrib.exe

cp /scratch/dmu/installer_wrf/WPS/namelist.wps .
cp /scratch/dmu/installer_wrf/WPS/util/plotgrids_new.ncl .

ln -sf /scratch/dmu/installer_wrf/WPS/ungrib/Variable_Tables/Vtable.ECMWF Vtable

cd ..
```

This step prepares all required WRF runtime files and WPS executables for data preprocessing.

---

## Step 3. Prepare the WRFDA Working Directory

Move into the **asimilasi_d01** directory and link all required WRFDA executables, libraries, lookup tables, and configuration files.

```bash
cd asimilasi_d01

ln -sf /scratch/dmu/installer_wrf/WRFDA/var/external/crtm_2.3.0 .
ln -sf /scratch/dmu/installer_wrf/WRFDA/var/da .
ln -sf /scratch/dmu/installer_wrf/WRFDA/var/da/da_radiance .
ln -sf /scratch/dmu/installer_wrf/WRFDA/var/run/radiance_info .

cp /scratch/dmu/installer_wrf/WRFDA/var/run/be.dat.cv3 be.dat

ln -sf /scratch/dmu/installer_wrf/WRFDA/var/build/da_wrfvar.exe .
ln -sf /scratch/dmu/installer_wrf/WRFDA/var/build/da_update_bc.exe .

ln -sf /scratch/dmu/installer_wrf/WRFDA/var/da/da_sound .
ln -sf /scratch/dmu/installer_wrf/WRFDA/run/*TBL .

cp /scratch/dmu/installer_wrf/WRFDA/var/test/update_bc/parame.in .
cp /scratch/dmu/installer_wrf/WRFDA/var/run/VARBC.in .
cp /scratch/dmu/installer_wrf/WRFDA/var/run/leapsec.dat .
cp /scratch/dmu/installer_wrf/WRFDA/var/test/radiance/namelist.input namelist.input

cp /scratch/dmu/installer_wrf/WRFDA/var/da/da_radar/da_radar_rf.inc .
cp /scratch/dmu/installer_wrf/WRFDA/var/da/da_radar/da_radial_velocity.inc .

ln -sf /scratch/dmu/installer_wrf/WRFDA/var/da/da_radar .

cd ..
```

This step prepares the complete WRFDA environment required for Rapid Update Cycle (RUC) data assimilation.

---

## Step 4. Create Additional Assimilation Domains

Duplicate the first assimilation directory for the remaining model domains.

```bash
cp -a asimilasi_d01 asimilasi_d02
cp -a asimilasi_d01 asimilasi_d03
```

The working directory now contains three independent WRFDA environments.

---

## Step 5. Prepare the ARWpost Environment

Move into the **ARWpost** directory and link the compiled executable, source files, scripts, and default namelist.

```bash
cd ARWpost

ln -sf /scratch/dmu/installer_wrf/ARWpost .
ln -sf /scratch/dmu/installer_wrf/ARWpost/src/ARWpost.exe .
ln -sf /scratch/dmu/installer_wrf/ARWpost/src .
ln -sf /scratch/dmu/installer_wrf/ARWpost/scripts .

cp /scratch/dmu/installer_wrf/ARWpost/namelist.ARWpost namelist.ARWpost
```

ARWpost is used to convert WRF output into visualization-ready formats.

---

## Expected Directory Structure

```
WRFRUC_training/
├── run_wrfruc/
├── asimilasi_d01/
├── asimilasi_d02/
├── asimilasi_d03/
└── ARWpost/
```

---

## Verification

Verify that the required symbolic links and copied files have been created successfully.

```bash
tree -L 2 WRFRUC_training
```

or

```bash
ls -l run_wrfruc
ls -l asimilasi_d01
ls -l ARWpost
```

If no errors are reported and the required executables are linked correctly, the working directory is ready for the next step.
