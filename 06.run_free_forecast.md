# 06. Run Free Forecast

## Overview

After completing the fourth Rapid Update Cycle (RUC), the radar observations have been assimilated at **00, 03, 06, and 09 UTC**. At this stage, the atmospheric initial conditions and lateral boundary conditions have been significantly improved by incorporating the latest radar observations.

The next step is to perform a **free forecast**, where the WRF model integrates forward in time without additional data assimilation until the next global model initialization becomes available.

For this tutorial, the final analysis at **09 UTC** is used as the initial condition for a **15-hour free forecast (09 until 00 UTC on the following day)**.

---

## High-Resolution Forecasts and the Butterfly Effect

For convection-permitting WRF simulations (approximately **1 km or finer**, such as the **333 m** domain used in this training), forecast errors grow rapidly because of the nonlinear nature of the atmosphere.

This phenomenon is commonly referred to as the **Butterfly Effect**, where very small uncertainties in the initial atmospheric state amplify with time through chaotic atmospheric dynamics.

Consequently,

- Small initial errors grow rapidly during model integration.
- Convective storms become increasingly difficult to predict after several hours.
- Forecast skill decreases much faster than for coarse-resolution global models.

For this reason, high-resolution operational forecasting systems generally employ **short-range forecasts** with **frequent cycling** using the latest global analysis.

A typical operational schedule is

| Global Analysis | Regional Forecast |
|-----------------|-------------------|
| 00 UTC | RUC + Free Forecast |
| 06 UTC | RUC + Free Forecast |
| 12 UTC | RUC + Free Forecast |
| 18 UTC | RUC + Free Forecast |

Rather than producing a single long forecast (48–72 hours), the model is restarted every **6 hours** using the latest global analysis (e.g., ECMWF IFS HRES) together with newly available radar observations. This strategy minimizes the growth of forecast errors and provides more accurate short-range predictions of convective weather.

## Step 1. Replace the WRF Initial Conditions

The fourth assimilation cycle produces the final analysis that will be used as the starting point of the free forecast.

Replace the original WRF initial conditions in the working directory with the updated analysis generated by **./da_wrfvar.exe**, and replace the lateral boundary condition for Domain 1 with the output generated by **da_update_bc.exe**.

```bash
cd /scratch/$USER/WRFRUC_training/asimilasi_d01

cp -rv wrfbdy_d01 \
/scratch/$USER/WRFRUC_training/run_wrfruc/

cp -rv wrfvar_output \
/scratch/$USER/WRFRUC_training/run_wrfruc/wrfinput_d01

cd /scratch/$USER/WRFRUC_training/asimilasi_d02

cp -rv wrfvar_output \
/scratch/$USER/WRFRUC_training/run_wrfruc/wrfinput_d02

cd /scratch/$USER/WRFRUC_training/asimilasi_d03

cp -rv wrfvar_output \
/scratch/$USER/WRFRUC_training/run_wrfruc/wrfinput_d03
```

The working directory should now contain

```text
wrfinput_d01
wrfinput_d02
wrfinput_d03
wrfbdy_d01
```

These files represent the final analyzed atmospheric state after four Rapid Update Cycles and are ready for the free forecast.

### Step 3.  Submit the wrf.exe

Copy the Slurm job script into the working directory.

```bash
cd /scratch/$USER/WRFRUC_training/run_wrfruc

cp -rv /scratch/dmu/training_data/script/run_wrf.sh .
```

The job run_wrf.sh script should contain the following configuration.

```bash
#!/bin/bash
#SBATCH --job-name=run_wrf
#SBATCH --partition=riset
#SBATCH --nodes=2
#SBATCH --ntasks-per-node=320
#SBATCH --time=03:00:00
#SBATCH --mem=720GB
#SBATCH --output=run_wrf_%j.out
#SBATCH --error=run_wrf_%j.err

source ~/.bashrc_wrf

module purge
ml restore wrf_module

source /software/compilers/intel-oneapi/2025.2.1/setvars.sh --force

time mpirun ./wrf.exe
```

Submit the forecast job.

```bash
sbatch run_wrf.sh
```

Monitor the queue.

```bash
squeue -u $USER
```

Monitor the WRF log.

```bash
tail -f rsl.out.0000
```
