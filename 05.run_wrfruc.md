# 05. Run Rapid Update Cycle (WRFDA)

## Objective

This module performs a **Rapid Update Cycle (RUC)** using the **WRF Data Assimilation (WRFDA)** system to improve the initial atmospheric state generated from the ECMWF IFS HRES forecast.

During the **12-hour spin-up period**, filtered weather radar observations from three radar sites are assimilated every **3 hours**:

- **C-Band Radar Jakarta**
- **X-Band Radar Cengkareng**
- **X-Band Radar Purwakarta**

Two radar observation types are assimilated:

- **Radar Reflectivity (dBZ)**
- **Radial Velocity (Vr)**

The cycling procedure continuously updates the WRF initial condition before each short-range forecast by incorporating the latest radar observations. This allows the model to better represent the location, intensity, and internal structure of convective systems.

The outputs generated in this module serve as the updated initial conditions for the subsequent **free forecast** simulation.

## Step 2. Real.exe
The **real.exe** program converts the meteorological fields generated by WPS (`met_em*`) into the initial and lateral boundary conditions required by WRF.
Copy the provided job real.sh script

```bash
cp -rv /scratch/dmu/training_data/script/real.sh .
```

The real.sh script should contain the following configuration.

```bash
#!/bin/bash
#SBATCH --job-name=real.exe
#SBATCH --partition=riset
#SBATCH --nodes=1
#SBATCH --ntasks-per-node=20
#SBATCH --time=03:00:00
#SBATCH --mem=720GB
#SBATCH --output=real_%j.out
#SBATCH --error=real_%j.err

source ~/.bashrc_wrf

module purge
ml restore wrf_module

source /software/compilers/intel-oneapi/2025.2.1/setvars.sh --force

time mpirun ./real.exe

```

Submit the real.exe job

```bash
sbatch real.sh
```

Monitor the queue 

```bash
squeue -u $USER
```

Monitor the execution log.

```bash
tail -f rsl.out.0000
```

The expected outputs are:

- `wrfinput_d01`
- `wrfinput_d02`
- `wrfinput_d03`
- `wrfbdy_d01`

These files provide the background (first guess) fields that will be updated during the Rapid Update Cycle using radar observations.
