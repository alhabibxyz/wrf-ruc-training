# 05. Run WPS

## Objective

The **WRF Preprocessing System (WPS)** prepares ECMWF IFS HRES data for the WRF model by converting the meteorological fields into model-ready input files.

During this process:

- **geogrid.exe** generates the geographical information for each WRF domain using the configured terrain, land-use, and static geographical datasets.
- **ungrib.exe** extracts meteorological variables from the ECMWF IFS HRES GRIB files and converts them into an intermediate WPS format.
- **metgrid.exe** horizontally interpolates the meteorological fields onto the WRF model domains and produces the final input files required by **real.exe**.

The outputs generated in this module will be used as the initial and boundary conditions for the Rapid Update Cycle (RUC) and subsequent WRF forecast simulations.

---
## Step 1. Enter the Working Directory

Navigate to the WRF-RUC working directory.

```bash
cd /scratch/$USER/WRFRUC_training/run_wrfruc
```

---

## Step 2. Copy the WRF Namelist Files

Copy the prepared **namelist.wps** into the working directory.

```bash
cp -rv /scratch/dmu/training_data/namelist/namelist_wrf/namelist.wps .
```

These files define the WRF domains, simulation period, geographical datasets, and preprocessing options.

---

## Step 3. Verify `namelist.wps`

Before running **geogrid.exe**, verify that the **namelist.wps** file has been configured correctly.

Open the file using your preferred text editor.

```bash
cat namelist.wps
```

Verify that the configuration matches the following example.

```bash
&share
 wrf_core         = 'ARW'
 max_dom          = 3
 start_date       = '2026-07-06_00:00:00','2026-07-06_00:00:00','2026-07-06_00:00:00',
 end_date         = '2026-07-07_00:00:00','2026-07-07_00:00:00','2026-07-07_00:00:00',
 interval_seconds = 10800
 io_form_geogrid  = 2
 debug_level      = 0
/

&geogrid
 parent_id         = 1, 1, 2
 parent_grid_ratio = 1, 3, 3
 i_parent_start    = 1, 116, 49
 j_parent_start    = 1, 99, 123
 e_we              = 310, 310, 385
 e_sn              = 310, 310, 385
 geog_data_res     = 'default','default+cglc_100m+srtm_jabar+srtm_banten','default+cglc_100m+srtm_jabar+srtm_banten'
 dx                = 3000
 dy                = 3000
 map_proj          = 'mercator'
 ref_lat           = -6.488
 ref_lon           = 106.810
 truelat1          = -6.488
 pole_lat          = 90
 pole_lon          = 0
 geog_data_path    = '/scratch/dmu/installer_wrf/WPS_GEOG'
 opt_geogrid_tbl_path = './geogrid/'
/

&ungrib
 out_format = 'WPS'
 prefix     = 'FILE'
/

&metgrid
 fg_name              = 'FILE'
 io_form_metgrid      = 2
 opt_metgrid_tbl_path = './metgrid'
/
```

Verify the following configuration before proceeding:

- **Simulation period** (`start_date` and `end_date`) matches the available ECMWF IFS HRES data.
- **Number of domains** (`max_dom`) is correctly defined.
- **Domain configuration** (`parent_id`, `parent_grid_ratio`, `e_we`, `e_sn`, etc.) matches the desired model setup.
- **High-resolution geographical datasets** are included in `geog_data_res`.
- **geog_data_path** points to the correct WPS geographical dataset directory.
- **opt_geogrid_tbl_path** points to the customized `GEOGRID.TBL.ARW`.

> [!NOTE]
> Any changes to the model domains, simulation period, or geographical datasets must be made **before** executing `geogrid.exe`. Once `geo_em.d0*.nc` files have been generated, `geogrid.exe` must be rerun if the `namelist.wps` configuration is modified.

## Step 4. Geogrid

Execute **geogrid.exe** to generate the geographical information for each model domain.

```bash
./geogrid.exe
```

Expected output:

```text
geo_em.d01.nc
geo_em.d02.nc
geo_em.d03.nc
```

> [!WARNING]
> If the following error appears

```text
./geogrid.exe:
error while loading shared libraries:
libnetcdf.so.22:
cannot open shared object file
```

the WRF libraries and runtime modules have not been loaded correctly.

Continue with the following steps.
### Step 4a. Configure the Runtime Environment

Create a shell configuration file containing the required WRF library paths.

```bash
cat > ~/.bashrc_wrf << 'EOF'
# =========================================
# WRF LIBRARIES
# =========================================

export DIR=/scratch/dmu/installer_wrf/libraries

export NETCDF=$DIR
export HDF5=$DIR
export JASPERLIB=$DIR/lib
export JASPERINC=$DIR/include

export PATH=$DIR/bin:$PATH

export LD_LIBRARY_PATH=$DIR/lib:$LD_LIBRARY_PATH
export LIBRARY_PATH=$DIR/lib:$LIBRARY_PATH

export CPATH=$DIR/include:$CPATH
export INCLUDE=$DIR/include:$INCLUDE
EOF
```

Load the environment.

```bash
source ~/.bashrc_wrf
```

Verify the environment variables.

```bash
echo $DIR
echo $NETCDF
echo $HDF5
echo $JASPERLIB
echo $JASPERINC
```
### Step 4b. Load the Runtime Modules

Load the Intel OneAPI runtime modules required by WRF.

```bash
module purge

module load oneapi/umf/0.11.0
module load oneapi/tbb/2022.2
module load oneapi/compiler-rt/2025.2.1
module load oneapi/compiler-intel-llvm/latest
module load oneapi/mpi/2021.16
```

Verify

```bash
module list
```

Save the module collection.

```bash
module save wrf_module
```

Restore the runtime environment.

```bash
source ~/.bashrc_wrf

module purge

ml restore wrf_module

source /software/compilers/intel-oneapi/2025.2.1/setvars.sh --force
```
Run **geogrid.exe** again.

```bash
./geogrid.exe
```

## Step 5. Ungrib

Create symbolic links to the converted ECMWF IFS HRES files.

```bash
./link_grib.csh \
/scratch/dmu/training_data/ecmwf_ifs/2026070600/converted/ecmwf* .
```

This command creates a sequence of

```text
GRIBFILE.AAA
GRIBFILE.AAB
...
```

Run ungrib

```bash
./ungrib.exe
```

Expected output

```text
FILE:2026-07-06_00
FILE:2026-07-06_01
...
```

These intermediate files will be used by **metgrid.exe**.

## Step 5. Metgrid

Since **metgrid.exe** supports parallel execution, submit the job through the Slurm scheduler.

Copy the provided job script.

```bash
cp -rv /scratch/dmu/training_data/script/metgrid.sh .
```

The script should contain the following configuration.

```bash
#!/bin/bash
#SBATCH --job-name=metgrid
#SBATCH --partition=riset
#SBATCH --nodes=1
#SBATCH --ntasks-per-node=20
#SBATCH --time=03:00:00
#SBATCH --mem=720GB
#SBATCH --output=metgrid_%j.out
#SBATCH --error=metgrid_%j.err

source ~/.bashrc_wrf

module purge
ml restore wrf_module


source /software/compilers/intel-oneapi/2025.2.1/setvars.sh --force


time mpirun ./metgrid.exe
```

Submit the job.

```bash
sbatch metgrid.sh
```

Monitor the queue.

```bash
squeue -u $USER
```

Monitor the execution log.

```bash
tail -f metgrid.log.0000
```

Expected output

```text
met_em.d01.*
met_em.d02.*
met_em.d03.*
```

These files contain the meteorological fields interpolated onto the WRF model domains and are ready for **real.exe** and the Rapid Update Cycle.
